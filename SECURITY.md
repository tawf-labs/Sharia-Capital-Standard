# Security Policy

## Reporting a Vulnerability

**DO NOT** open a public issue for security vulnerabilities.

Please report security vulnerabilities to: security@sharia-capital-standard.org

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and provide a timeline for fixes.

## Audit Status

This project is currently **NOT AUDITED**. Do not use in production without:
1. Independent security audit
2. Sharia Supervisory Board review
3. Comprehensive testing in testnet environments

## Security Considerations

### Smart Contract Risks
- Reentrancy attacks
- Integer overflow/underflow
- Access control vulnerabilities
- Upgradeability risks

### Sharia Compliance Risks
- Guaranteed return mechanisms
- Fixed-yield structures
- Prohibited asset exposure
- Governance bypass
